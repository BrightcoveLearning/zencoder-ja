    解像度の設定

    このトピックには、ビデオ解像度に関連するエンコード設定の詳細が含まれています。

    概要

    1つまたは複数の解像度設定を使用して、出力ビデオの解像度を制御できます。

    1.どちらかを設定することから始めますサイズ、幅、または高さ。widthそしてheight優先するsize。これらのいずれも設定しない場合、元のビデオ解像度が保持されます。

    2.次に、入力がターゲット出力よりも小さい場合にビデオのサイズを大きくするかどうかを決定します（高級）。デフォルトでは、ビデオをアップスケールしません。多くの場合、ビデオプレーヤーはビデオを問題なくアップスケールできるため、エンコードの時点でビデオをアップスケールする必要はありません（ファイルサイズが大きくなるだけです）。

    3.最後に、アスペクト比へのアプローチを選択します。入力ビデオのアスペクト比が選択した解像度と異なる場合は、元のアスペクト比を保持することができます。ターゲットのアスペクト比に合うようにビデオを引き伸ばします。ターゲットのアスペクト比に一致するように入力をトリミングします。または、ターゲットのアスペクト比に一致するようにビデオを黒いバーで埋めます。

    注：多くのビデオ形式では、幅と高さの値を偶数にする必要があります。出力形式でこれが必要な場合、Zencoderは幅、高さ、またはその両方を切り捨てて、必要に応じてこの要件を満たします。

    size

    size:String

    APIバージョン：V1、V2

    親：出力

    有効な値：WxH、ここでWとHはそれぞれ幅と高さです。両方の次元は、4で割り切れる整数である必要があります。

    互換性のあるジョブタイプ：すべて

    例：480x320

    説明:

    640×480や1280×720のようにWxHで表される出力ファイルの解像度。ご了承ください幅そして高さサイズよりも優先されます。

    アスペクトモードはデフォルトで保持されるため、入力ビデオが16：9で、4：3の出力解像度が要求された場合、出力は16：9のアスペクト比を維持します。これにより、出力寸法が要求されたサイズと一致しなくなります。使用するアスペクトモードこれを制御します。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "size": "480x320"
    }
  ]
}

    関連項目：幅、高さ、およびアスペクトモード

    幅

    width:Integer

    APIバージョン：V1、V2

    親：出力

    有効な値：4で割り切れる整数。

    互換性のあるジョブタイプ：すべて

    例：480

    説明:

    ビデオフレーム幅。幅が指定されていない場合は、元の幅を使用するか、スケールしてサイズまたは高さ設定。

    幅と高さ優先するサイズ。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "width": "480"
    }
  ]
}

    関連項目：サイズそして高さ

    高さ

    height:Integer

    APIバージョン：V1、V2

    親：出力

    有効な値：4で割り切れる整数。

    互換性のあるジョブタイプ：すべて

    例：640

    説明:

    ビデオフレームの高さ。高さが指定されていない場合は、元の高さを使用するか、次のようにスケーリングしますサイズまたは幅設定。

    ご了承ください幅と高さが優先されますサイズ。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "height": "640"
    }
  ]
}

    関連項目：サイズそして幅

    高級な

    upscale:Boolean

    APIバージョン：V1、V2

    親：出力

    デフォルト： false

    有効な値：正しいか間違っているか

    互換性のあるジョブタイプ：すべて

    例： true

    説明:

    デフォルトでは、Zencoderは増加しませんサイズに一致する入力ビデオの（または「高級」）幅そして高さ APIリクエストで指定します。一般に、ビデオをアップスケーリングするとサイズが大きくなるだけですが、実際には品質は上がりません。再生デバイス（Flash、HTML5ビデオ、iPhoneなど）は、エンコーダーに処理させるのではなく、ビデオのサイズを大きくすることができます。

    ただし、Zencoderに入力ビデオのサイズを強制的に拡大させたい場合は、「アップスケール」をtrueに設定します。その場合、出力解像度よりも小さい入力ビデオは、解像度に合わせて拡大されます。たとえば、出力仕様が480×360で、誰かが320×240のビデオを送信した場合、このオプションをtrueに設定すると、ビデオは480×360にアップサイズされ、それ以外の場合は320×240のままになります。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "upscale": true
    }
  ]
}

    aspect_mode

    aspect_mode:String

    APIバージョン：V1、V2

    親：出力

    デフォルト：保存する

    有効な値：保存、ストレッチ、トリミング、またはパッド

    互換性のあるジョブタイプ：すべて

    例：作物

    説明:

    入力のアスペクト比が要求された出力アスペクト比と一致しない場合、出力解像度はどのようになりますか？ 「aspect_mode」はこの決定を制御します。幅と高さの両方が変更される可能性があることに注意してください。

    例として、入力ファイルが1280×720（ワイドスクリーン16：9）であるが、ターゲット出力が640×480（標準4：3）であると仮定します。この問題には4つの基本的なアプローチがあり、Zencoderはそれらすべてをサポートしています。

    • 保存する：デフォルトでは、Zencoderは元のファイルのアスペクト比を保持するため、ワイドスクリーンコンテンツを送信して標準解像度を要求した場合、出力ファイルはワイドスクリーンのアスペクト比を維持し、出力サイズ内に収まります。おそらく、プレーヤーはフレームを埋めるために黒いバーを追加します。この例では、出力ファイルは640×360（640×480のターゲット内に収まるワイドスクリーン16×9）になります。
    • ストレッチ：アスペクトモードが「ストレッチ」の場合、出力ビデオは、画像が歪んでいても、要求された幅と高さに正確に一致します。したがって、この例では、出力は640×480になりますが、垂直方向に引き伸ばされたように見えます。注意：これは、小さいビデオを大きいフレームサイズに拡大するかどうかを決定する上記の「アップスケール」オプションと同じではありません。
    • 作物：このオプションは、上/下または左/右からピクセルをトリミングすることにより、出力サイズに一致するようにビデオに「ズームイン」するようにZencoderに指示します。したがって、入力がワイドスクリーン1280×720で、出力ターゲットが標準の640×480の場合、入力ファイルの左右から約160ピクセルをトリミングし（4：3ムービーを作成）、サイズを変更します。 640×480まで。ワイドスクリーン以外のVHS（私たちのような）を覚えるのに十分な年齢の場合、これは古い「パンアンドスキャン」オプションです。
    • パッド：パッドオプションは、出力フレームサイズに正確に一致するようにビデオをレターボックス化するようにZencoderに指示します。ターゲットプレーヤーがビデオのパディング方法を知らない場合は、このオプションを使用します。それ以外の場合は、実際のファイルに黒いバーを含めることでビットを無駄にしているだけです。この例に従うと、このオプションは1280×720（16：9）のコンテンツを取得し、640×360のムービーを作成し、60ピクセルの黒いバーを上部と下部に配置することで640×480（4：3）に変換します。映画。したがって、映画は640×480になり、上部と下部に黒いバーが表示されます。

    アスペクトモードでは、幅と高さ提供されている（またはサイズ）;それ以外の場合、1つのディメンションのみが指定されている場合は、他のディメンションを比例して計算します。

    注：VP6 video_codec「パッド」aspect_modeをサポートしていません。VP6ビデオで「パッド」を要求すると、デフォルトの「保存」モードが使用されます。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "aspect_mode": "crop"
    }
  ]
}

    sample_aspect_ratio

    sample_aspect_ratio:String

    APIバージョン：V2

    親：出力

    互換性のあるジョブタイプ：すべて

    例：8：9

    説明:

    出力ビデオに使用されるサンプル（ピクセル）アスペクト比。サイズ、幅、高さ、aspect_modeモードのオプションはすべて、表示サイズではなく、エンコードされたピクセルサイズを参照します。

    出力ビデオのサンプル/ピクセルに使用するアスペクト比を設定します。これは、「8：9」、「32:27」、またはそれぞれ1〜65535の2つの正の整数のその他の比率として指定されます。

    この設定に任意の（有効な）値を指定すると、デフォルトの「aspect_mode」が「preserve」から「stretch」に変更されます。つまり、デフォルトでは、エンコードされた出力の幅/高さも指定する必要があります。

    注:予測可能な出力解像度を得るには、「upscale」オプションをこのオプションと組み合わせて使用すると便利な場合があります。

    このオプションは、H.264およびHEVCコーデックを使用する場合にのみ適用されます。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "sample_aspect_ratio": "8:9"
    }
  ]
}

    force_interlace_headers

    force_interlace_headers:Boolean

    APIバージョン：V2

    親：出力

    互換性のあるジョブタイプ：すべて

    デフォルト： false

    説明:

    一部のデコーダーはインターレースコンテンツを要求しますが、プログレッシブコンテンツは問題なく再生されます。この設定により、プログレッシブコンテンツのメタデータに「インターレース」ヘッダーが追加され、実際にインターレースされているように見えます。

    注:一部のメディア検査ツールはこれにだまされず、ヘッダー情報にもかかわらず、このコンテンツをプログレッシブとして報告します。）

    このオプションは、H.264コーデックを使用する場合にのみ適用されます。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "force_interlace_headers": true
    }
  ]
}
