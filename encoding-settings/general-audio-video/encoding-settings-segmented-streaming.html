    セグメント化されたストリーミング設定

    このトピックには、セグメント化されたストリーミングに関連するエンコード設定の詳細が含まれています。

    概要

    Zencoderは、さまざまな形式のアダプティブストリーミングとセグメント化ストリーミングをサポートしています。

    • HTTPライブ（セグメント化）ストリーミングとプレイリスト
    • マイクロソフトスムーズストリーミング
    • ハイウィンズSMIL

    各ストリーミング形式は、以下にリストされているAPIオプションの異なるサブセットを使用します。詳細については、各ストリーミングタイプのガイドを参照してください。

    注：一部のプレーヤーは、オーディオストリームとビデオストリームの長さがセグメントの長さを超えて異なる場合を処理しません。この問題が発生した場合は、短いセグメント長を使用してみてください。

    注意事項

    • ジョブには、少なくとも1つの標準出力またはセグメント化された出力が必要です。プレイリスト出力のみでジョブを作成することはできません。
    • プレイリスト出力は、を除いてほとんどの出力オプションを無視しますurl、base_url、filename、streams、public、およびtype。
    • セグメント化された出力を一時ストレージにアップロードすると、 ZIP アーカイブとしてアップロードされます。

    streaming_delivery_format

    streaming_delivery_format:String

    APIバージョン：V1、V2

    親：outputs

    有効な値：ダッシュ

    互換性のあるジョブタイプ：すべて

    例：ダッシュ

    説明:

    必要なマニフェスト、ディレクトリ構造、またはその他のパッケージを含む、特定のストリーミング構成を使用して配信される出力のフォーマット/プロトコルを設定します。

    'dash'出力の場合、これによりデフォルトのsegment_secondsが2に変更され、固定のキーフレーム間隔が適用され、MPDマニフェストが含まれ、初期化ファイルとセグメントファイルを含むディレクトリ構造が作成されます。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "streaming_delivery_format": "dash"
    }
  ]
}

    関連項目： Streaming_delivery_profile

    streaming_delivery_profile

    streaming_delivery_profile:String

    APIバージョン：V1、V2

    親：outputs

    デフォルト：住む

    有効な値：ライブ、hbbtv_1.5、on_demand

    互換性のあるジョブタイプ：すべて

    例：住む

    説明:

    ストリーミング配信形式のプロファイルを設定し、プロファイルとの互換性のためにオプションが選択されていることを確認します。有効なオプションは、選択したものによって異なりますstreaming_delivery_format。

    にとってdash出力、現在サポートされているプロファイルは次のとおりです。

    • ライブ-設定しますDASHプロファイルを「urn：mpeg：dash：profile：isoff-live：2011」に設定し、単一のSegmentTemplateそれぞれについてAdaptationSet。DRMの場合、8バイトのIVを使用します。PlayReadyメディアにPSSHアトムを含むバージョン1.1を使用します。
    • hbbtv_1.5-を設定しますDASHプロファイルを「urn：hbbtv：dash：profile：isoff-live：2012」に設定し、オーディオおよびビデオ表現に必要な属性を含めます。DRMの場合、16バイトのIVを使用します。PlayReady PSSHアトムを除いて、バージョン1.0を使用します。
    • on_demand-を設定しますDASHプロファイルを「urn：mpeg：dash：profile：isoff-on-demand：2011」に設定し、メディアをセグメント化するのではなく、ストリームごとに単一の断片化されたファイルとしてエンコードします。それ以外はライブプロファイルと同じです。
    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "streaming_delivery_format": "dash",
      "streaming_delivery_profile": "hbbtv_1.5"
    }
  ]
}

    関連項目：streaming_delivery_format

    segment_seconds

    segment_seconds:Float

    APIバージョン：V1、V2

    親：outputs

    デフォルト：10

    有効な値：1.0から3600.0までの数値。 2.0未満の値は、次の場合にのみ許可されますbyte_range_segmenting有効になっています。非常に短いセグメントは再生の問題を引き起こすことがわかっているため、最終的なセグメントは指定された値より長くなる可能性があります。

    互換性のあるジョブタイプ：すべて

    例：8

    説明:

    セグメント化された出力の各セグメントの最大期間を設定します。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "segment_seconds": 8
    }
  ]
}

    alternate_audio

    alternate_audio:Hash

    APIバージョン：V1、V2

    親：outputs

    有効な値：代替オーディオグループのハッシュ。それぞれがオーディオストリームレンディションハッシュの配列です。

    互換性のあるジョブタイプ：すべて

    説明:

    の代替オーディオストリームのセットを提供しますHLSプレイリスト。必要ですmax_hls_protocol_version 4以上になります。

    このオプションは挿入しますEXT-X-MEDIAプレイリストにタグを付けます。場合nameが設定されていない場合、指定されたものに基づいて名前が生成されますlanguage。

    代替オーディオレンディションの有効なオプション

    • language：「en」、「es」、「zh」などのオーディオトラックの言語のISO639コード。
    • name（オプション）：代替オーディオグループの名前。名前が設定されていない場合、デフォルトで言語名になります。
    • path（オプション）：このオーディオレンディションのM3U8プレイリストへのオプションのパス。パスが設定されていない場合、この代替オーディオストリームは、ソースストリームに含まれるオーディオを記述します。
    • source（オプション）：同じジョブのメディア出力のラベル。これを使用して、このオーディオストリームを使用するレンディションのコーデックを設定できます。
    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "playlist",
      "alternate_audio": {
        "custom_audio_group": [
          {
            "source": "english-audio",
            "path": "alternate-audio-en.m3u8",
            "language": "en"
          },
          {
            "source": "french-audio",
            "path": "alternate-audio-fr.m3u8",
            "language": "fr"
          }
        ],
        "secondary_audio_group": {
          "path": "path/to/secondary-audio-group.m3u8",
          "language": "en",
          "name": "Secondary English Audio Track",
          "source": "secondary-english-audio"
        }
      }
    }
  ]
}

    },

    prepare_for_segmenting

    prepare_for_segmenting:String or Array

    APIバージョン：V1、V2

    親：outputs

    有効な値：hls、mss、dash

    互換性のあるジョブタイプ：すべて

    例：hls,dash

    説明:

    出力をトランスマックスする予定の場合HLS、MSS、および/またはDASHキャプション付きでMP4にエンコードした後、このオプションをtrueに設定して、セグメント化のキャプションとキーフレームタイミングを含めます。これにより、セグメント化されたバージョンに、トランスマックスにもかかわらずHLS互換のキャプションを含めることができ、一貫した時間間隔でのセグメント化が可能になります。

    Zencoderが必要に応じてキャプションとキーフレームを準備できるように、この出力を後でトランスマックスして作成するセグメント化された出力のタイプを指定します。

    注:現在、キャプションを追加することはできませんHLSトランスマックス時に出力します。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "prepare_for_segmenting": "hls,dash"
    }
  ]
}

    generate_keyframe_manifest

    generate_keyframe_manifest:Boolean

    APIバージョン：V2

    親：outputs

    デフォルト：false

    有効な値：trueまたはfalse

    互換性のあるジョブタイプ：すべて

    例：true

    説明:

    を生成しますHLS早送りおよび巻き戻し再生に必要なキーフレーム（Iフレーム）マニフェスト。

    デフォルトのキーフレームマニフェストファイル名はプレイリストファイル名と同じですが、「-iframe-index」が追加されています（したがって、「master.m3u8」プレイリストの場合、キーフレームマニフェストは「master-iframe-index.m3u8」になります）。キーフレームマニフェストファイル名は、次のコマンドで上書きできます。keyframe_manifest_filename。

    次の場合、デフォルトで有効になりますmax_hls_protocol_versionまたはhls_protocol_version 4以上に設定されています。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "type": "segmented",
      "format": "ts",
      "generate_keyframe_manifest": true,
      "keyframe_manifest_filename": "my_iframe_index.m3u8"
    }
  ]
}

    keyframe_manifest_filename:String

    allow_skipped_sources

      "allow_skipped_sources": true,
      "streams": [
