    CAE 設定

    このトピックでは、Zencoder のコンテキスト認識エンコーディング (CAE) 設定について説明します。

    概要

    コンテキスト対応エンコーディングでは、指定したガイドラインに基づいて、ビデオ自体の特性に基づいて、ソースビデオの複数のレンディションが作成されます。これにより、ソースの特性を考慮すると、基本的に価値のないレンディションを作成することで、動画のトランスコーディングと保存にかかるコストを回避できます。

    を参照してくださいCAEガイド設定に関する追加情報と推奨事項については。

    video_codecs

    video_codecs:Array

    APIバージョン：V2

    親：dynamic_profile_options

    有効な配列項目の値：h264、hevc

    デフォルト値：なし

    互換性のあるジョブタイプ：VOD

    例：["h264", "hevc"]

    説明:

    混合コーデック出力の場合、レンディションを作成するコーデックの配列。現在、許可されているアイテムの値は次のとおりです。h264そしてhevc。

    {
    "dynamic_profile_options": {
      "max_renditions": 6,
      "video_codecs" : ["h264", "hevc"'],
      "video_codec_options": {
        "h264": {
          "min_renditions": 2,
          "max_resolution": { "width": 1280, "height": 720 }
        },
        "hevc": {
          "min_renditions": 2,
          "max_resolution": { "width": 1920, "height": 1080 }
        }
      }
    }
  }

    video_codec_options

    video_codecs:Object

    APIバージョン：V2

    親：dynamic_profile_options

    デフォルト値：なし

    互換性のあるジョブタイプ：VOD

    例：

    "video_codec_options": {
  "h264": {
    "min_renditions": 2,
    "max_resolution": { "width": 1280, "height": 720 }
  },
  "hevc": {
    "min_renditions": 2,
    "max_resolution": { "width": 1920, "height": 1080 }
  }
}

    説明:

    混合コーデック出力の場合、各コーデックのオプションを指定するオブジェクト。各プロパティは、名前の付いたオブジェクトですh264またはhevc、そのプロパティは、そのコーデックのレンディションの制約を設定します。これらのオブジェクトに使用できるプロパティは次のとおりです。

    • min_renditions必須
    • min_bitrate
    • max_bitrate
    • max_first_rendition_bitrate
    • min_resolution
    • max_resolution
    • max_first_rendition_resolution
    注:これらの設定は、グローバルレベルでも使用されます。グローバルレベルとコーデック単位の両方の設定を使用する場合は、値に互換性がある必要があります。たとえば、グローバルレベルで設定されたmax_renditionsは、コーデックレベルで設定されたmin_rendition値の合計以上である必要があります。
    {
    "dynamic_profile_options": {
      "max_renditions": 6,
      "video_codecs" : ["h264", "hevc"'],
      "video_codec_options": {
        "h264": {
          "min_renditions": 2,
          "max_resolution": { "width": 1280, "height": 720 }
        },
        "hevc": {
          "min_renditions": 2,
          "max_resolution": { "width": 1920, "height": 1080 }
        }
      }
    }
  }

    min_renditions

    min_renditions:Integer

    APIバージョン：V2

    親：dynamic_profile_options

    有効な値：1-15

    デフォルト値：1

    互換性のあるジョブタイプ：VOD

    例：3

    説明:

    生成するレンディションの最小数。

            {
          "input": "s3://zencodertesting/test.mov",
          "generate_dynamic_profile": true,
          "dynamic_profile_options": {
            "min_renditions": 2
          }
        }

    APIバージョン：V2

    quality_rate_tune

    quality_rate_tune:Number

    APIバージョン：V2

    親：dynamic_profile_options

    有効な値： -40.0-40.0

    デフォルト値：0

    互換性のあるジョブタイプ：VOD

    例：2

    説明:

    設定することによりquality_rate_tune 1から40までの正の数にすると、CAEプロファイルのビデオ品質は高くなりますが、ビットレートが高くなります。同様に、-1から-40の範囲の数値が与えられると、ビデオ品質を低下させる代わりに、ビットレートを下げることができます。

    たとえば、-10を使用すると、ビットレートが約10％低くなることが期待できます。10の場合、ビットレートは一般に約10％増加します。の正の値のCAEは、これらの数値は概算です。quality_rate_tune非常に高いビデオ品質を確保するために特別な予防措置を講じます。また、粒度やレンディションの数など、ビットレートを制限する他のパラメータが存在する場合があります。

    一般的なコンテンツと必要なビデオ品質と帯域幅は大きく異なるため、さまざまな値を試してみることをお勧めします。 

            {
          "input": "s3://zencodertesting/test.mov",
          "generate_dynamic_profile": true,
          "dynamic_profile_options": {
            "min_renditions": 2,
            "max_renditions": 10,
            "quality_rate_tune": 2
          }
        }

    select_baseline_profile_configuration

    select_baseline_profile_configuration:Boolean

    APIバージョン：V2

    親：dynamic_profile_options

    デフォルト値： true

    互換性のあるジョブタイプ：VOD

    例：100年

    説明:

    プロファイルで使用される少なくとも1つのレンディションは、ベースラインプロファイルになります。select_baseline_profile_configurationはデフォルトでtrueであり、512x288以下の解像度を得るには「少なくとも1つのレンディション」が必要であることに注意してください。

            {
          "input": "s3://zencodertesting/test.mov",
          "generate_dynamic_profile": true,
          "dynamic_profile_options": {
            "min_renditions": 2,
            "max_renditions": 10,
            "select_baseline_profile_configuration": true
          }
        }

    Additional settings

    以下は、CAEに使用できる追加の一般的なエンコーディング設定です。

    追加の設定
    名前 タイプ 説明 デフォルト
    speed 整数 エンコード速度の設定。 3
    one_pass ブール値 1パスエンコーディングを強制します。 false
    tuning 文字列 コーデック固有のチューニングオプション。  
    start_clip 文字列/数値 特定の時間から開始して、元のビデオのサブクリップを作成します。これは、次の形式のタイムコードにすることができます。HH:MM:SS.S、または開始する秒数を表す小数。 0
    clip_length 文字列/数値 特定の長さのサブクリップを作成します。これは、次の形式の期間にすることができますHH:MM:SS.S、またはクリップの長さを秒単位で表す小数。 ソース期間

    HDR設定

    CAEはHDR10もサポートしており、次の設定を含めることができますdynamic_profile_options。

    追加の設定
    名前 タイプ 説明 デフォルト
    master_display オブジェクト HDR出力のカラーボリュームと輝度設定を含むハッシュ。 なし
    max_content_light_level 整数 コンテンツ全体の最大光レベル1平方メートルあたりのニットまたはカンデラ（cd / m2）-整数値。 なし
    max_frame_average_light_level 整数 平方メートルあたりのコンテンツフレームニットまたはカンデラの最大平均光レベル（cd / m2）-整数値。10000までの値が許可されますが、100〜200の範囲の値が一般的に最良の結果を生成します。 なし

    ビデオ構成設定

    CAEが出力に必要な特定の構成を生成していない場合は、この機能を使用することをお勧めします。

    これは、CAEが出力の一部として生成する必要がある/生成する必要があるビデオ解像度とコーデック設定を記述するオブジェクトの配列です（上記の例）。この配列はオプション。省略した場合、192x108からコンテンツの解像度までの範囲のデフォルト解像度の配列がCAEのデフォルト入力セットとして使用されます。

    ビデオ構成設定
    名前 タイプ 説明 必須/任意
    width 整数 ビデオフレーム幅[ピクセル] 必須
    height 整数 ビデオフレームの高さ[ピクセル] 必須
    sample_aspect_ratio 整数：整数ペア サンプルのアスペクト比（例：1：1、4：3） オプショナル
    frame_rate 数値 ビデオフレームレート[fps] オプショナル
    video_codec_profile 文字列 ビデオコーデックプロファイル オプショナル
    video_codec_level 文字列 ビデオコーデックレベル オプショナル
    video_reference_frames 整数 参照フレーム数の制約。デフォルト= 4（H.264ベースラインの場合は3）。 オプショナル
    video_bframes 整数 Bフレーム数の制約。デフォルト= 3（H.264ベースラインの場合は0）。 オプショナル
    required ブール値 これをtrueに設定すると、出力で推奨される構成設定が保証されます。デフォルト= false。 オプショナル
