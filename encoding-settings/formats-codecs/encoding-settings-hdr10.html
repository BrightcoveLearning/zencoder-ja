    HDR 設定

    このトピックには、HDRビデオ出力に関連するエンコード設定の詳細が含まれています。

    概要

    ZencoderがサポートするHDR形式の詳細と、HDR形式パラメーターの推奨事項については、 HDRビデオガイド。

    keep_hdr_metadata

    master_display:Boolean

    APIバージョン：V2

    親：出力

    デフォルト： false

    互換性のあるジョブタイプ： VOD

    説明:

    trueに設定すると、このパラメーターは、入力ファイルから（ほとんどまたはすべての）HDR関連のメタデータをインポートして出力に渡すようにZencoderに指示します。

    たとえば、HDR10、PQ10、またはHLG10形式のHEVCエンコードビデオストリームのトランスコーディングは、次のジョブリクエストを使用して実行できます。 

    {
  "input": "s3://mybucket/hdr_master.mp4",
  "outputs": [
    {
     "video_codec": "hevc",
     "video_codec_profile": "main10",
     "video_bit_depth": 10,
     "preserve_hdr_metadata": true
    }
  ]
}

    このパラメーターは、HDR関連のメタデータを直接指定して、他のパラメーターと組み合わせて使用することもできます。たとえば、以下のJSONジョブリクエストはその方法を示していますpreserve_hdr_metadataパラメータは持ち越しにも使用できますmaster displayそしてcontent light level HEVCでエンコードされたDolbyVision8.1ストリームからのメタデータ、DolbyVision動的メタデータ（RPUファイル）は、dolby_vision_rpu_urlパラメーターを使用して個別に通信されます。

    {
  "input": "s3://mybucket/dv81_master.mp4",
  "outputs": [
    {
     "video_codec": "hevc",
     "video_codec_profile": "main10",
     "video_bit_depth": 10,
     "hdr_format": "dv8.1",
     "preserve_hdr_metadata": true,
     "dolby_vision_rpu_url": "s3://mubucket/dv81_master_rpu.rpu"
    }
  ]
}

    参照：

    • HDRフォーマットの概要
    • HDRフォーマットパラメータの指定
    • CAEでHDRフォーマットパラメータを指定する

    hdr_format

    hdr_format:String

    APIバージョン：V2

    親：outputs、dynamic_profile_options

    有効な値：hdr10、pq、hlg、dv5、dv8.1

    デフォルト：なし

    互換性のあるジョブタイプ： VOD

    説明:

    出力の生成に使用されているHDRフォーマットのタイプを指定します。

    このパラメーターは、入力ファイルにHDR形式関連のメタデータが含まれていない場合に使用する必要があります。したがって、形式タイプと関連するすべてのメタデータの両方を直接指定する必要があります。

    たとえば、J2KでエンコードされたMFXとして提供されるPQカラーグレーディングされたコンテンツは、次のジョブ要求によってPQ10出力ストリームに変換できます。

    {
  "input": "s3://mybucket/j2k_master.mfx",
  "outputs": [
    {
     "video_codec": "hevc",
     "video_codec_profile": "main10",
     "video_bit_depth": 10,
     "hdr_format": "pq"
    }
   ]
}

    または。同じJ2KMXFマスターがHDR10コンテンツを表し、その内容を説明する追加ファイルが付属している場合マスターディスプレイそしてコンテンツの光レベルメタデータパラメータ、次にHDR10 HEVCストリームでのエンコードは、次のジョブリクエストを使用して実行できます。

    {
  "input": "s3://mybucket/j2k_master.mfx",
  "outputs": [
    {
      "video_codec": "hevc",
      "video_codec_profile": "main10",
      "video_bit_depth": 10,
      "hdr_format": "hdr10",
      "master_display": {
        "red": {"x": 0.708, "y": 0.292},
        "green": {"x": 0.17, "y": 0.797},
        "blue": {"x": 0.131, "y": 0.046},
        "white_point": {"x": 0.31273,"y": 0.329},
        "luminance": {"max": 1000,"min": 0.0001}
      },
      "max_content_light_level": 1000,
      "max_frame_average_light_level": 180
    }
  ]
}

    参照：

    • HDRフォーマットの概要
    • HDRフォーマットパラメータの指定
    • CAEでHDRフォーマットパラメータを指定する

    master_display

    master_display:Object

    APIバージョン：V2

    親：outputs、dynamic_profile_options

    デフォルト：なし

    互換性のあるジョブタイプ： VOD

    説明:

    ビデオコンテンツのマスタリングに使用されたディスプレイのカラーボリューム（カラープライマリ、ホワイトポイント、および輝度範囲）を表す構造。この構造で使用される特定の値は、によって定義されます。 SMPTE ST 2086標準。

    マスターディスプレイのメタデータは必須であり、HDR-10およびDolbyVision8.1形式でビデオをエンコードするときに出力に対して実行する必要があります。

    以下の例は、マスター表示メタデータを指定する方法を示しています。

    {
  "input": "s3://mybucket/j2k_master.mfx",
  "outputs": [
    {
      "video_codec": "hevc",
      "video_codec_profile": "main10",
      "video_bit_depth": 10,
      "hdr_format": "hdr10",
      "master_display": {
        "red": {"x": 0.708, "y": 0.292},
        "green": {"x": 0.17, "y": 0.797},
        "blue": {"x": 0.131, "y": 0.046},
        "white_point": {"x": 0.31273,"y": 0.329},
        "luminance": {"max": 1000,"min": 0.0001}
      },
      "max_content_light_level": 1000,
      "max_frame_average_light_level": 180
    }
  ]
}

    max_content_light_level

    max_content_light_level:Number

    APIバージョン：V2

    親：outputs、dynamic_profile_options

    デフォルト：なし

    有効な値：0-10000

    互換性のあるジョブタイプ： VOD

    例：1000

    説明:

    ビデオシーケンス全体の最大ピクセル（および赤、緑、青のチャネル）ごとの光レベルを記述します。平方メートルあたりのカンデラで表されます。この値は0に設定できます。これは、そのような境界が不明であるか、課されていないことを意味します。このパラメータは、別名MaxCLLによって定義されますCAE 861.3標準。

    以下の例は、 max_content_light_level、およびmax_frame_average_light_levelメタデータを指定できます：

  "input": "s3://mybucket/j2k_master.mfx",

    {
      "video_codec": "hevc",
      "video_codec_profile": "main10",
      "video_bit_depth": 10,
      "hdr_format": "hdr10",
      "master_display": {
        "red": {"x": 0.708, "y": 0.292},
        "green": {"x": 0.17, "y": 0.797},
        "blue": {"x": 0.131, "y": 0.046},
        "white_point": {"x": 0.31273,"y": 0.329},
        "luminance": {"max": 1000,"min": 0.0001}
      },
      "max_content_light_level": 1000,
      "max_frame_average_light_level": 180
    }
  ]
}

    video_color_range

    video_color_range:String

    APIバージョン：V2

    親：outputs、dynamic_profile_options

    有効な値：limited、full

    デフォルト：なし

    互換性のあるジョブタイプ： VOD

    説明:

    ピクセル値のダイナミックレンジを示します。たとえば、8ビットピクセルの場合、limited Y（luma）ピクセル値が16から235の範囲にとどまる一方で、full Y（ルマ）ピクセル値が0から255の範囲にあることを意味します。

    コンテンツの大部分は現在、を使用して作成およびエンコードされていますlimitedピクセル。しかしながらfull -範囲ピクセルも使用される場合があります。たとえば、10ビット幅のフルレンジピクセルはDolbyVision5形式で使用されます。

    使用例： 

    {
  "input": "s3://mybucket/dv5_master.mp4",
  "outputs": [
   {
    "video_codec": "hevc",
    "video_codec_profile": "main10",
    "video_bit_depth": 10,
    "hdr_format": "dv5",
    "video_color_range": "full",
    "dolby_vision_rpu_url": "s3://mubucket/dv5_master_rpu.rpu"
   }
  ]
}
