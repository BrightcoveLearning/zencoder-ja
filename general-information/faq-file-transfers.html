よくある質問：ファイル転送

このトピックでは、エンコードのためにファイルをZencoderに転送し、出力のために宛先にファイルを転送することに関する質問に答えます。

よくある質問

  • サポートしている転送プロトコルは何ですか？HTTP？FTP？

    入力:HTTP、HTTPS、FTP、SFTP、Rackspace クラウドファイル、Google Cloud Storage、Azure、Amazon S3 をサポートしています。

    出力:FTP、SFTP、Rackspace クラウドファイル、Azure、Google Cloud Storage、Amazon S3 をサポートしています。現在、一般的なHTTPアップロードはサポートされていません。

    クラウドファイルのユーザー:使用するリージョン（DFW、ORD、または英国）は、のようにプロトコルに追加することで指定できますcf+ord://username:api_key@container/object。現在、サポートされているクラウドファイルリージョンは、分散ファイアウォール、ORD、英国のみです。デフォルトでは、分散分散ファイアウォール領域が使用されます。

    別の転送方法が必要ですか？お知らせください。

  • トランスコーディング用の元のファイルはどこに置くことができますか？

    私たちは、HTTP、HTTPS、FTP、またはSFTPを介してどこからでもファイルを引き込むことができます。また、Azure コンテナ、Google クラウドストレージ、または S3 バケット（プライベートバケットを含む）、および Rackspace クラウドファイルからファイルをプルすることもできます。

    例:

    http://example.com/path/to/file.avi
http://s3.amazonaws.com/bucket/file.avi
s3://bucket/file.avi
gcs://bucket/file.avi
cf://username:api_key@container/file.avi
azure://account-name:account-key@container/file.avi
https://example.com/path/to/file.avi
ftp://example.com/path/to/file.avi
sftp://example.com/path/to/file.avi
    ポートに関する注意：ファイルアドレスにポートを含める必要がある場合は、次のように、URLのドメインの最後でポートを宣言する必要があります。 
    sftp://example.com:8888/path/to/file.avi

  • ファイルを安全に転送できますか？

    確かに。SFTP または HTTPS を使用し、次のようにユーザ名とパスワードを URL に含めます。

    
https://user:pass@example.com/file.mp4
sftp://user:pass@example.com/file.mp4

  • プライベート S3、Google クラウドストレージ、またはクラウドファイルバケットからファイルをプルできますか？

    はいZencoder アカウントにアクセス資格情報を追加するか、Zencoder のバケットへの読み取りアクセスを付与できます。

  • ファイルサイズの制限はありますか？

    なしファイルサイズの制限はありません。実際、お客様が最高のコピーを送信するように促すために、大きな入力動画には追加料金を請求しないことを選択しました。あなたが私たちに送信するビデオの品質が高いほど、私たちはあなたに与えることができる出力品質が高くなります。

    ただし、ソースビデオの再生時間には 24 時間の制限があります。

  • 完成した動画はどこに行きますか？

    動画を送信する場所にはいくつかのオプションがあります。

    ファイルを FTP または SFTP サーバー、S3 または Google クラウドストレージバケット、または Rackspace クラウドファイルに送信できます。詳細については、API ドキュメントの url オプションを参照してください。

    追加することを忘れないでください資格情報またZencoderにバケットへのアクセスを許可する S3またはGoogleCloudStorageを使用している場合。

    または、出力先を指定しないと、24 時間ファイルが保持されます。動画をダウンロードするために使用できる URL がお客様に提供されます。24 時間後に動画を削除しますので、ダウンロードすることを忘れないでください。

    API バージョン 2 以降、サーバーにファイルをアップロードできない場合は、代わりに S3 にアップロードし、すべてのファイルの URL を提供します。

  • エンコードされたファイルを S3 に送信する場合、ACL のアクセス許可を設定するにはどうすればよいですか？

    私たちのAPIを使用してください。詳細については、 access_controlの API ドキュメントを参照してください。

    • ファイル転送

    Zencoder は Amazon EC2 インスタンス上で動作するため、ファイルトランスコーディングとライブトランスコーディングには広い IP アドレスが使用されています。Amazon は定期的に新しい範囲をリストに追加します。この範囲は AWS IP 範囲 API で確認できます。

    通知

    通知は現在から送信されていますがsg-77f03012、これはいつでも変更される可能性があります。これを説明するために、X-Zencoder-Notification-Secret配信されたすべての通知にヘッダーを追加しました。これは APIダッシュボードで確認できます。このダッシュボードでは、現在のシークレットを確認し、必要に応じて新しいシークレットを生成できます。

    このヘッダーはすべてのリクエストとともに配信され、Zencoder アカウントから通知が配信されたことを保証するために使用できます。注 :通知を安全に保つために、HTTPS エンドポイントに通知を配信してください！

    注:サーバーは新しいデータセンターに移行する場合があり、IP アドレスは予告なく変更されることがあります。

  • ビデオが載っているハードドライブを Zencoder に出荷できますか？

    エンコードする動画が多数あるがインターネット経由で転送したくない場合は、Amazon の AWS Import/Export を使用できます。動画を含むハードドライブを Amazon に発送し、Amazon がお客様のために S3 にアップロードします。

    Amazonでは、ドライブの出荷準備や送付先など、簡単な手順が用意されています。
