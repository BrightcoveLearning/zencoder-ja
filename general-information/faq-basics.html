    よくある質問:基本事項

    このトピックは、Zencoderに関するいくつかの基本的な質問に答えます。

    よくある質問

    • 何をZencoder行う？

      Zencoderはオンラインビデオ変換/エンコーディング/トランスコーディングサービスです。ウェブサイトのユーザーやその他のソースから取得した動画を取得し、ウェブ再生、携帯電話、その他のサポートが必要なデバイスと互換性のある形式に変換します。

    • 誰がZencoderにとって？

      Zencoder独自のトランスコーディングサーバーを構築、保守、スケーリングするという面倒で時間のかかるプロセスを経ることなく、ビデオをアプリケーションに統合したい開発者向けです。ユーザーからのビデオアップロードを受け入れるWebサイトまたはアプリケーションには、何らかの形式のトランスコーディングバックエンドが必要です。ユーザーがアップロードした動画のサイズや形式が、ウェブやその他のデバイスで再生できることはめったにありません。

    • エンコードするビデオをアップロードするにはどうすればよいですか？

      の目的Zencoder Webフォームから手動でアップロードすることなく、ビデオを自動的に変換することです。これを達成するために、 APIこれにより、アプリケーションはZencoder新しいビデオをエンコードする必要があるときに警告します。まず、ビデオはファイルサーバーまたはAmazonS3やRackspaceCloudFilesなどのクラウドホスティングサービス上にある必要があります。Zencoder Web経由でアクセスできます。その後、あなたは私たちを使用することができますリクエストビルダー指示するエンコーディングジョブリクエストを作成しますZencoderビデオの場所、エンコード方法、および結果の出力ビデオの配置場所。を参照してくださいリクエストビルダー詳細については、アカウントで。

    • を使用してアップロードできるファイルのサイズに制限はありますか？リクエストビルダー？

      はい-リクエストビルダーを使用してアップロードできるのは、サイズが5GB未満のファイルのみです。

    • エンコーディングジョブとは何ですか？

      エンコードジョブは、入力ビデオまたはオーディオファイルを取得し、それを1つ以上の出力形式に変換するプロセスです。を介してエンコードジョブを開始できますAPIまたはを使用してリクエストビルダーあなたのアカウントで。エンコードプロセスでは、Zencoder最初に、ホストされている場所から入力ファイルをダウンロードします。次に、ファイルを指定された出力設定に変換し、出力ファイルを独自のサーバーまたは設定した特定の場所（多くの場合、入力ファイルと同じホスティングサーバー）にアップロードします。

    • プログラミング言語とはZencoderと互換性がありますか？

      HTTPリクエストを送信できる任意の言語。これには、Python、Ruby、Java、.NET（C＃、VB、F＃など）、PHP、Perl、Javascript、Tcl、Haskell、Factor、Common Lisp、Smalltalk、Forth、ML、R、S、J、K、 Bash、Clojure、Scala、Modula2、Modula3、ColdFusion、Emacs Lisp、Lua、C、C ++、D、Objective-C、NewtonScript、Mathematica、PL / SQL、Cobol、Arc、Assembly。ここに希望の言語が表示されない場合は、インターネットをサポートしているかどうかを確認してください。もしそうなら、それはおそらく大丈夫です。

      言い換えれば、Zencoder APIは、出入りするHTTPリクエストです。いくつかの統合ライブラリを提供しているので、少し使いやすくなっていますが、統合するのはそれほど難しくありません。Zencoderライブラリなし。組み込みのJSONまたはXMLサポートはボーナスですが、JSONは非常に単純なので、JSONをサポートしていない言語（Cobol？）を使用した場合は、おそらく自分でハックすることができます。

    • どのような統合ライブラリを提供していますか？

      統合するためのライブラリをいくつか作成しましたZencoderより簡単に：

      • ルビー
      • PHP
      • エリクサー
      • Node.js
      • Python
      • Java

      一部のお客様は、独自のライブラリを作成し、それらを使用できるようにしています。

      • C＃.Net
      • Clojure
      • 行く

      今後、ライブラリをさらに追加したいと考えています。ライブラリを構築している場合は、お知らせください。ドキュメントに追加します。

    • 統合ライブラリが必要ですか？

      いいえ。少なくとも、Zencoder JSONまたはXML本文を使用してHTTPリクエストを送信しているだけです。あなたはからこれを行うことができますcurlを使用したコマンドライン。入手する201 Created戻って、あなたはちょうど統合しましたZencoder。

      それ以外にも、HTTP経由でoutput_notificationを受け取りたい場合があります。URLをお知らせください。ジョブのステータスをPOSTでお知らせします。JSONまたはXML本文を含むHTTPPOSTを受信するように設定されたサイトのURLが必要です。

      最後に、APIを使用してエンコーディングジョブで他のことをする、working_with_jobs、working_with_jobs、working_with_jobsなど。これもHTTPリクエストです。

    • 統合ライブラリは何をしますか？

      当社の統合ライブラリは、生のHTTPリクエストからユーザーを抽象化します。使用するRubyライブラリたとえば、次のようなジョブを作成できます。

      Zencoder::Job.create({:input => 's3://bucket/key.mp4',
    :outputs => [{:label => 'vp8 for the web',
    :url => 's3://bucket/key_output.webm'}]})

      このようなことをするのではなく：

      require 'uri'
    require 'json'
    response = Net::HTTP.post_form(URI.parse('jobs.json'),
    {:input => 's3://bucket/key.mp4',
    :outputs => [{:label => 'vp8 for the web',
    :url => 's3://bucket/key_output.webm'}]}.to_json)

      ライブラリは、APIを直接操作するときに持っていない機能を有効にしません。彼らはそれを少し簡単にします。

    • 私の言語の統合ライブラリがない場合はどうすればよいですか？

      3つのオプションがあります。

      • 独自に作成してリリースします
      • ライブラリなしで作業する
      • 私達に聞いてくれ作品にある場合

    • 自分で作ることはできますかZencoderライブラリとそれを共有しますか？

      もちろん。オープンソースの場合Zencoderライブラリ、おそらくそれにリンクするでしょう。お気軽にコピーしてくださいRubyライブラリ選択した言語で。

    • 開発マシンで通知を受け取るにはどうすればよいですか？

      たとえば、サードパーティのツールを使用できますngrok。

    • 統合モードとは何ですか？

      統合モードは、アカウントの統合をテストできるようにするアカウントの設定です。Zencoderエンコーディングの料金はかかりません。あなたはそれについてもっと読むことができますテストジョブと統合モードセクション。

    • プライバシーモードとは何ですか？

      プライバシーモードでは、特定のAPIパラメータを適用して、コンテンツを不正なビューから保護します。また、システムで取得されたサムネイルを無効にし（出力サムネイルを指定することもできます）、機密情報の可能性を難読化します。Zencoder従業員は、いかなる理由でもプライベートファイルを表示しません。

      アダルトコンテンツを送信する場合Zencoderアカウントはプライバシーモードである必要があります。

    • APIリクエストのレートは制限されていますか？

      はい、Zencoder特定の時間枠内で特定のメソッドを呼び出すことができる回数を制限します。制限は、出力ごとに追跡される進行状況要求を除いて、メソッド（リソース）ごとに追跡されます。たとえば、ジョブと出力情報を取得するための呼び出しはすべて同じ制限にカウントされ、出力進捗の呼び出しは出力ごとに個別に追跡されます。

      各アカウントは、1 分あたり 60 リクエストが許可されます。ただし、ジョブ作成リクエストは 1 分あたり 1,000 リクエストに制限され、出力進行状況要求は 1 分あたり 60 リクエストに制限されます。

      各HTTP応答には、「」というヘッダーが含まれています。X-Zencoder-Rate-Remaining "。このヘッダーは、現在の時間枠内で特定のリソースに対して行うことができる呼び出しの数をリストします。

      割り当てを超えた場合、Zencoder "の本文で403エラーを返します403 Forbidden (Rate Limit Exceeded) "。また、追加のヘッダー「」を送り返します。Retry-After "、クォータがリセットされるまでの秒数が含まれます。

    • APIリクエストのレートが制限されているのはなぜですか？

      1人のお客様が確実に使用できるように、APIリクエストを適切な数に制限しますZencoder他のお客様に悪影響を与えることはありません。レート制限なしでは、あるお客様がシステムを使用できないような方法で、当社のシステムに要求が溢れる可能性があります。これは、顧客が最初にAPIで作業しているときに偶然に定期的に発生します。単純なプログラミングエラーは、顧客が送信する意味のないリクエストでAPIをフラッディングする可能性があります。

      アプリケーションが適切に設定されている限り、レート制限は問題になりません。本文に「レート制限を超えました」と403を返した場合は、指定された期間後にリクエストを再試行してください。

      これらの制限を定期的に実行している場合は、セットアップとオプションについてお問い合わせください。
